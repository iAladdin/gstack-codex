# Review

1. Read `../references/principles.md`.
2. Read `../references/review.md`.

## Rules

- Lead with findings, not summary.
- Prioritize bugs, regressions, missing tests, security, and performance.
- Include precise evidence and file references.
- If no material findings exist, say that explicitly and mention residual risk.
