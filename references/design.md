# Design Playbook

Use this reference for `plan-design-review`, `design-consultation`, and `design-review`.

## Design Review Checklist

- Does the page clearly communicate the main action?
- Is the hierarchy obvious in five seconds?
- Does the UI feel intentional rather than template-default?
- Are spacing, typography, and color choices consistent?
- Are loading, empty, error, and success states designed?
- Does the mobile layout feel first-class, not compressed desktop?

## Design Consultation Output

When asked to create or refine a design direction, provide:

1. Product tone and visual intent
2. Type choices and hierarchy rules
3. Color system
4. Component rules
5. Motion guidance
6. Screen-level guidance for the immediate task

## Design Review With Code

When making fixes:

- Preserve the existing design system unless the user asked for a reset.
- Change the smallest set of files that materially improves the experience.
- Verify responsive behavior after the edit.
