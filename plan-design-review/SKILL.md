# Plan Design Review

1. Read `../references/principles.md`.
2. Read `../references/planning.md`.
3. Read `../references/design.md`.

## Focus

- clarity of the main action
- hierarchy and scanability
- interaction cost
- responsiveness
- states: empty, loading, error, success
- distinctiveness vs template-default

If the plan is vague, rewrite the weak parts instead of giving generic praise.
